== Generate a Backend Service (SVC)

The generator supports three flavors of projects: *angular*, *with NgRx* and *without NgRx*. +
It is **recommended to use the NgRx** flavor, because it leads to better structured projects and the state management is already in place. +
The creation of a new workspace takes a while, as the generator sets up a complete Angular application with all necessary dependencies and configurations.

Releases of generator:
https://github.com/onecx/onecx-svc-generator/releases

.Generate the SVC with following commands
Download the latest release of the generator from GitHub and save it as `onecx-svc-generator.jar` in your desired directory. +
[source,bash]
----
curl -L -o onecx-svc-generator.jar \
https://github.com/onecx/onecx-svc-generator/releases/download/v0.1.1/onecx-svc-generator.jar
----
Use generator with following command
[source,bash]
----
java -jar onecx-svc-generator.jar create-svc \
  --name onecx-demo-svc \
  --group org.tkit.onecx \
  --package org.tkit.onecx.demo \
  --build true
----
flag `--build true` will build the application after generation. If you want to build it manually, you can omit this flag and run `mvn clean install` in the generated directory.
flag `--name` is used to specify the name of the generated service, which will be used as base for project name, artifact name and some essential application parts as following:
flag `--group` is used to specify the groupId of the generated service, which will be used as base for package name and some essential application parts as following:
flag `--package` is used to specify the base package name of the generated service, which will be used as base for package name and some essential application parts as following

The generator creates a new directory with name from flag name and sets up the OneCX basic structure for dao, controller, api and service. +
Please note that the product name from your command is not used as base of entity names, but it is used as base for some essential application parts as following:
* `onecx-<name>` +
  Project name +
  Remote name for module federation +
  Helm chart name for deployment and microservice registration in OneCX
* `org.tkit.onecx.<package>` +
  Package name for the generated service
